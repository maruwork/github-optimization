# GitHub Optimization Productization Goal

Status: Complete

## Goal

Turn this shelf from a single checklist note into a reusable common tool that can be applied to pre-public repositories with explicit placement rules.

## Complete When

- the tool has a clear `README.md`
- reusable templates and checklists exist under explicit shelves
- root files, `.github/` files, GitHub settings, and optional expansion items are separated
- project application rules are explicit and consistent with the local shared-tool installation rule
- the shared entry surface can route readers to the tool

## Out Of Scope

- applying the tool to every repository
- enabling GitHub settings on hosted repositories
- rewriting existing repository-local community files

## Failure Conditions

- the tool remains a single mixed checklist
- readers cannot tell what belongs in root, `.github/`, or GitHub settings
- the tool can only be used by re-reading prose instead of following a bounded route
