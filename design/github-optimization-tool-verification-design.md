# GitHub Optimization Tool Verification Design

Status: Complete

## GOTV-T1 Inventory

### Read

- `README.md`

### Write

- working notes and final summary only

### Do

- list every named external tool
- list every named GitHub-native feature

### Accept

- nothing named in the shelf is left unclassified

## GOTV-T2 External Tool Verification

### Read

- current official or primary documentation for each external tool

### Write

- `README.md`
- summary log

### Do

- verify whether the tool still exists and is maintained enough to recommend
- verify the intended role described in the shelf
- note when the tool is optional or superseded

### Accept

- recommendation text is grounded in current primary information

## GOTV-T3 GitHub-Native Verification

### Read

- current GitHub documentation for each native feature

### Write

- `README.md`
- checklists

### Do

- verify feature availability and intended use
- avoid claiming a route that no longer matches GitHub's current surface

### Accept

- GitHub-native guidance matches current GitHub documentation

## GOTV-T4 Shelf Alignment

### Read

- verified notes
- `README.md`
- checklist files

### Write

- `README.md`
- `checklists/*.md`

### Do

- classify items as `use`, `conditional`, or `do not recommend`
- remove overbroad or stale claims
- add replacement guidance where necessary

### Accept

- the shelf is usable as a current decision aid

## GOTV-T5 Summary

### Write

- a durable tool-verification summary note

### Do

- record verified items
- record classification and rationale

### Accept

- the next maintainer can see what was verified and why
